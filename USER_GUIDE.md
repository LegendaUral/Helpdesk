# Руководство использования HelpDesk

## 👤 Для сотрудника (user)

### Шаг 1: Вход в систему
1. Открыть `http://localhost:5000`
2. Нажать на поле **Email** и ввести электронную почту
3. Нажать на поле **Пароль** и ввести пароль
4. Нажать кнопку **Вход**

### Шаг 2: Просмотр своих заявок
- На главной странице видны **только ваши заявки**
- Каждая заявка показывает:
  - 📌 Номер (ID)
  - 📝 Название
  - 🔴 Приоритет (красный = высокий, жёлтый = средний, зелёный = низкий)
  - 🟠 Статус (синий = новая, жёлтый = в работе, зелёный = решена, серый = закрыта)

### Шаг 3: Создание новой заявки
1. Нажать кнопку **+ Новая заявка**
2. Заполнить форму:
   - **Название** (обязательно) — краткое описание проблемы
   - **Описание** (обязательно) — подробное описание
   - **Приоритет** — выбрать из трёх вариантов
3. Нажать **Создать заявку**
4. Система создаст заявку со статусом **Новая**

### Шаг 4: Общение в чате заявки
1. Открыть заявку (кликнув на неё)
2. Внизу находится **Чат**
3. Ввести сообщение в поле и нажать кнопку **Отправить** ✈️
4. Сообщение появится в чате
5. Обновление чата происходит автоматически каждые 3 секунды

### Шаг 5: Отслеживание статуса
- **Новая** — только что создана, ждёт назначения специалиста
- **В работе** — специалист сейчас работает над проблемой
- **Решена** — проблема решена, специалист заканчивает
- **Закрыта** — заявка полностью закрыта, чат недоступен

### Выход
- Нажать на аватар/имя в верхнем углу
- Выбрать **Выход**

## 👨‍💼 Для специалиста поддержки (support)

### Шаг 1: Вход
- Те же действия, что и сотрудник

### Шаг 2: Просмотр очереди заявок
На главной странице видны:
- 📌 **Новые заявки** (без исполнителя)
- 📌 **Ваши заявки** (назначенные вам)

Специалист **НЕ ВИДИТ** заявки, назначенные коллегам.

### Шаг 3: Поиск и фильтрация
1. Нажать **Поиск**
2. Фильтровать по:
   - 👤 **Исполнитель** — выбрать специалиста
   - 📅 **От даты** до **До даты** (формат: YYYY-MM-DD)
   - ↕️ **Сортировка** — по дате, приоритету или статусу
3. Нажать **Поиск**

### Шаг 4: Взятие заявки в работу
1. Открыть заявку из очереди
2. Внизу справа нажать **Взять в работу**
3. Статус заявки изменится на **В работе**
4. Вы становитесь исполнителем

### Шаг 5: Работа с заявкой
1. Прочитать описание проблемы
2. Написать комментарий в чате (если нужны уточнения)
3. Работать над решением
4. Обновлять статус по мере прогресса:
   - **В работе** — текущее состояние
   - **Решена** — когда проблема решена
   - **Закрыта** — финальное состояние

### Шаг 6: Изменение статуса
1. В разделе **Изменить статус** выбрать новый статус
2. Нажать **Сохранить**
3. Если статус **Закрыта**, чат становится неактивным

### Шаг 7: Отпуск заявки (если нужно)
- Если взяли заявку ошибочно:
  1. Нажать **Отпустить заявку**
  2. Статус вернётся на **Новая**
  3. Заявка вернётся в общую очередь

### Отличия от сотрудника:
- ❌ **Не может** создавать заявки
- ✅ **Может** менять статусы
- ✅ **Может** брать заявки в работу
- ✅ **Видит** все неназначенные заявки

## 👨‍💻 Для администратора (admin)

### Полные права
Администратор может делать всё, что может сотрудник и специалист, + управление пользователями.

### Управление пользователями

#### Просмотр активных пользователей
1. Нажать **Пользователи** в меню
2. Видны все активные пользователи с ролями и датами создания

#### Создание нового пользователя
1. Нажать **+ Новый пользователь**
2. Заполнить форму:
   - **Имя** (обязательно)
   - **Email** (обязательно, уникальный)
   - **Пароль** (обязательно)
   - **Роль** — выбрать:
     - 👤 **Сотрудник** (user)
     - 👨‍💼 **Специалист** (support)
     - 👨‍💻 **Администратор** (admin)
3. Нажать **Создать пользователя**

#### Редактирование пользователя
1. В списке нажать на пользователя
2. Изменить поля (все кроме email, если нужно)
3. Нажать **Сохранить**

#### Удаление пользователя
1. В списке нажать на пользователя
2. Нажать **Удалить**
3. Пользователь помечается как удалённый (мягкое удаление)
4. Его открытые заявки закрываются
5. Данные остаются в БД (для истории)

#### Просмотр удалённых пользователей
1. Нажать **Удалённые пользователи**
2. Видны все удалённые пользователи
3. Их нельзя восстановить из интерфейса (требуется БД)

### Управление заявками

#### Просмотр всех заявок
- На главной странице **видны ВСЕ заявки** (не только свои)
- Можно фильтровать и искать

#### Изменение статуса
1. Открыть заявку
2. В разделе **Изменить статус** выбрать новый статус
3. Нажать **Сохранить**
4. Статус обновится

#### Назначение исполнителя
1. Открыть заявку
2. В разделе **Назначить исполнителя** выбрать специалиста
3. Нажать **Назначить**
4. Заявка назначена выбранному специалисту

## 🔍 Фильтры и сортировка

### На главной странице (для всех)

#### Сортировка
- **По дате (новые)** — сначала новые заявки
- **По дате (старые)** — сначала старые заявки
- **По приоритету** — сначала высокий приоритет
- **По статусу** — сначала новые, потом в работе, решены, закрыты

### На странице поиска

#### Фильтры
- **Исполнитель** — текстовый поиск по имени
- **От даты** — дата начала (формат: 2024-01-15)
- **До даты** — дата конца (формат: 2024-01-31)

#### Результаты
- Показываются заявки, соответствующие фильтрам
- Применяется выбранная сортировка

## 💬 Работа с чатом

### Добавление сообщения
1. Открыть заявку
2. Найти раздел **Чат** внизу
3. Ввести текст в поле **"Введите сообщение..."**
4. Нажать кнопку **Отправить** ✈️

### Просмотр сообщений
- Сообщения обновляются автоматически каждые 3 секунды
- Цветовая разметка сообщений:
  - 🔵 **Синий** — обычный сотрудник (user)
  - 🔷 **Светло-синий** — специалист (support)
  - 🟠 **Оранжевый** — администратор (admin)

### Ограничения
- Чат доступен только пока заявка **открыта**
- Если статус **Закрыта** — чат неактивен
- Писать могут: автор, исполнитель и администратор

### Правила
- 📝 Сообщение не может быть пусто
- ⏰ Сообщения сохраняются с временной меткой
- 👤 Видно имя и роль отправителя

## 📱 Использование на мобильном

### Установка приложения

#### На Android (Chrome)
1. Открыть `http://[IP-адрес]:5000`
2. Нажать **меню** (три точки)
3. Выбрать **"Установить приложение"**
4. Подтвердить
5. Иконка появится на рабочем столе

#### На iOS (Safari)
1. Открыть `http://[IP-адрес]:5000` в Safari
2. Нажать кнопку **"Поделиться"** (стрелка вверх)
3. Выбрать **"На экран 'Домой'"**
4. Нажать **"Добавить"**
5. Иконка появится на рабочем столе

### Использование
- Все функции работают так же, как на рабочем столе
- Приложение работает в оффлайне (частично) благодаря Service Worker
- Оффлайн функции: просмотр кеша, базовая навигация
- Для полной функциональности нужен интернет

## 🆘 Частые вопросы и решения

### Я забыл пароль
- Обратитесь к администратору
- Администратор может пересоздать вашу учётную запись

### Я не вижу свою заявку
- **Если вы сотрудник**: видны только ваши заявки
- **Если вы специалист**: видны ваши + неназначенные заявки
- **Если вы администратор**: должны видеть все заявки
- Проверьте фильтры на странице поиска

### Я не могу создать заявку
- Специалист поддержки **не может** создавать заявки
- Это функция только для сотрудников и администраторов

### Почему чат не обновляется?
- Обновление происходит каждые 3 секунды
- Проверьте подключение к интернету
- Обновите страницу (F5)

### Я хочу отпустить заявку
1. Откройте заявку
2. Нажмите **Отпустить заявку** (красная кнопка)
3. Заявка вернётся в очередь других специалистов

### Как изменить свой пароль?
- Функция недоступна в текущей версии
- Обратитесь к администратору (он пересоздаст учётную запись)

### Почему я удалён?
- Администратор удалил вашу учётную запись
- Ваши открытые заявки автоматически закрыты
- Обратитесь к администратору для восстановления

## ⚙️ Настройка системы (для администратора)

### Первоначальная настройка
1. При первом открытии `http://localhost:5000` откроется страница **Setup**
2. Заполнить данные администратора:
   - Имя
   - Email
   - Пароль
   - Подтверждение пароля
3. Нажать **Создать администратора**
4. Система готова к работе

### Создание других администраторов
1. Перейти в **Пользователи**
2. Нажать **+ Новый пользователь**
3. Выбрать роль **Администратор**
4. Завершить создание

### Безопасность
- 🔐 Пароли хранятся в виде хеша SHA256
- 🔐 Сессии проверяются перед каждым запросом
- 🔐 Email должны быть уникальны
- 🔐 Первого администратора нельзя удалить

## 🐛 Отладка проблем

### Приложение не запускается
```bash
# Проверьте, что Python установлен
python --version

# Установите зависимости
pip install -r requirements.txt

# Запустите приложение
python app.py
```

### Ошибка: "Порт 5000 занят"
```bash
# Windows
netstat -ano | findstr :5000
taskkill /PID <PID> /F

# Linux/Mac
lsof -i :5000
kill -9 <PID>
```

### База данных повреждена
```bash
# Удалите и пересоздайте БД
rm instance/helpdesk.db
python app.py
```

### Не помните первого администратора
```bash
# Сотрите БД и создайте нового админа
rm instance/helpdesk.db
python app.py
# Откройте http://localhost:5000/setup
```

## 📚 Дополнительная информация

- Все даты в системе в формате: **DD.MM.YYYY HH:MM**
- Временная зона: **UTC+6 (Екатеринбург)**
- Email используется как уникальный идентификатор пользователя
- Удаление пользователей — мягкое (данные остаются в БД)
- Заявки永远не удаляются (только закрываются)
