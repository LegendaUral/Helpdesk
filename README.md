# HelpDesk — Система регистрации заявок в службе технической поддержки

## 📋 Описание

**HelpDesk** — это веб-приложение для управления заявками технической поддержки. Система позволяет сотрудникам создавать заявки на поддержку, специалистам отслеживать и обрабатывать их, а администраторам управлять пользователями и контролировать весь процесс.

### Ключевые возможности:
- 📝 Создание и отслеживание заявок
- 👥 Система ролей (администратор, специалист поддержки, сотрудник)
- 💬 Встроенный чат для коммуникации между автором и исполнителем
- 🔍 Поиск и фильтрация заявок
- 📱 PWA (Progressive Web App) — работает на мобильных устройствах
- 🗄️ SQLite база данных с надёжной архитектурой

---

## 🛠️ Стек технологий

- **Backend**: Python 3.8+ / Flask 3.0
- **Язык шаблонов**: Jinja2
- **База данных**: SQLite (через Flask-SQLAlchemy)
- **Frontend**: Bootstrap 5, HTML5
- **PWA**: Service Worker для работы в режиме оффлайн

## 👥 Роли пользователей

| Роль    | Описание | Права |
|---------|---------|-------|
| **user** | Сотрудник | • Создавать заявки<br>• Просматривать свои заявки<br>• Писать в чат своих заявок |
| **support** | Специалист поддержки | • Всё как user, плюс:<br>• Менять статус заявок<br>• Поиск заявок<br>• Брать заявку в работу<br>• Отпускать заявку |
| **admin** | Администратор | • Всё как support, плюс:<br>• Управление пользователями<br>• Назначение исполнителей<br>• Просмотр всех заявок<br>• Восстановление удалённых пользователей |

## 📊 Архитектура приложения

### Слой приложения (`app.py`)
- Инициализация Flask и SQLAlchemy
- Проверка удалённых пользователей перед каждым запросом
- Регистрация всех маршрутов
- Фильтры для форматирования дат
- Контекстные процессоры для передачи словарей в шаблоны

### Модели БД (`models.py`)

#### User (Пользователь)
```
id (int)           — первичный ключ
name (str)         — имя пользователя
email (str)        — уникальный email (для входа)
password_hash (str) — SHA256 хеш пароля
role (str)         — роль: 'user', 'support', 'admin'
created_at (dt)    — дата создания
is_deleted (bool)  — флаг удаления (мягкое удаление)
```

#### Ticket (Заявка)
```
id (int)           — первичный ключ
title (str)        — заголовок заявки
description (txt)  — описание проблемы
priority (str)     — 'low', 'medium', 'high'
status (str)       — 'new', 'in_progress', 'resolved', 'closed'
created_at (dt)    — дата создания
updated_at (dt)    — дата последнего изменения
author_id (fk)     — автор заявки (сотрудник)
executor_id (fk)   — исполнитель заявки (специалист)
```

#### Message (Сообщение в чате)
```
id (int)           — первичный ключ
ticket_id (fk)     — заявка, к которой относится сообщение
user_id (fk)       — автор сообщения
text (txt)         — текст сообщения
created_at (dt)    — дата отправки
```

### Маршруты и API

#### `routes/auth.py` — Аутентификация
- `GET/POST /setup` — первоначальная настройка (создание admin)
- `GET/POST /login` — вход в систему
- `GET /logout` — выход из системы

#### `routes/tickets.py` — Управление заявками
- `GET /` — список заявок (с фильтрацией по ролям)
- `GET/POST /tickets/new` — создание заявки
- `GET /tickets/<id>` — просмотр заявки и чата
- `POST /tickets/<id>/message` — добавить сообщение
- `POST /tickets/<id>/status` — изменить статус (support/admin)
- `POST /tickets/<id>/executor` — назначить исполнителя (admin)
- `POST /tickets/<id>/take` — взять заявку (support)
- `POST /tickets/<id>/release` — отпустить заявку (support)

#### `routes/users.py` — Управление пользователями (только admin)
- `GET /users` — список активных пользователей
- `GET /users/deleted` — список удалённых пользователей
- `GET/POST /users/new` — создание пользователя
- `GET/POST /users/<id>/edit` — редактирование пользователя
- `POST /users/<id>/delete` — удаление пользователя (мягкое удаление)

#### `routes/search.py` — Поиск и фильтрация
- `GET /tickets/search` — поиск по исполнителю и дате

#### `routes/api.py` — REST API
- `GET /api/messages/<ticket_id>` — загрузить сообщения в формате JSON
- `GET /api/tickets` — загрузить список заявок в JSON

## 🔐 Безопасность

- **Аутентификация**: сессии Flask с проверкой на каждый запрос
- **Хеширование паролей**: SHA256
- **Контроль доступа**: проверка ролей в `auth_required()`
- **Мягкое удаление**: удалённые пользователи помечаются флагом `is_deleted`, данные сохраняются
- **CSRF защита**: используется в формах через Jinja2

## 🌍 Доступ через интернет (с мобильного интернета)

### Способ 1: ngrok (быстро, временно)
```bash
ngrok http 5000
```
Ngrok выдаст публичный адрес вида `https://abc123.ngrok-free.app` — он открывается с любого устройства.

**Минус**: бесплатный ngrok даёт случайный адрес при каждом перезапуске.

- Арендуйте VPS (~200-400 руб/мес)
- Разверните приложение на VPS
- Используйте фиксированный адрес `https://yourdomain.com`

```bash
cloudflared tunnel --url http://localhost:5000
```

## 📝 Примеры использования

### Сценарий 1: Создание заявки сотрудником
1. Вход в систему (email/пароль)
2. Кнопка «Новая заявка»
3. Заполнить название, описание, выбрать приоритет
4. Отправить → заявка в статусе **Новая**

### Сценарий 2: Обработка заявки специалистом
1. Вход в систему (специалист support)
2. Видит заявки: новые и те, что ему назначены
3. Берёт заявку кнопкой «Взять в работу» → статус **В работе**
4. Пишет в чат комментарии/уточнения
5. Решает проблему, меняет статус на **Решена** → **Закрыта**

### Сценарий 3: Управление пользователями (администратор)
1. Вход в систему (администратор)
2. Меню → «Пользователи»
3. Создать/редактировать/удалить пользователя
4. Назначить исполнителя для заявки
